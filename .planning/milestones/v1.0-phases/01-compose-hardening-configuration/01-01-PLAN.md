## Phase Goal

**As an** operator, **I want to** `docker compose up` a hardened Coder stack that connects to Postgres on a host bind mount and is reachable at my configured access URL, **so that** I can run a production-trustworthy server whose data survives container recreation.

<objective>
Walking Skeleton slice: transform the upstream proof-of-concept `compose.yaml` into a production-grade stack and add the `.gitignore` that protects secrets. After this plan, `docker compose up -d` brings up Postgres on a host bind mount (one real write/persist path), Coder connects to it, the Coder `/healthz` healthcheck passes, and the UI is reachable at `CODER_ACCESS_URL` — the thinnest working end-to-end stack.

Purpose: Establish the persistent, pinned, health-gated, env-driven compose stack that Plan 02 (operator docs) and Phases 2-3 (backup scripts, workspace template) all build on. This is the architectural backbone recorded in SKELETON.md.
Output: Hardened `compose.yaml`; new `.gitignore`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| git repo → public/remote | Committed files may leak secrets if `.env` is tracked |
| host network → Postgres | Postgres must not be reachable from the host network |
| operator browser/proxy → Coder :7080 | Untrusted HTTP ingress crosses into the control plane |
| supply chain → Coder image | Image tag determines what code runs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | `.env` (DB password) committed to git | mitigate | `.gitignore` blocks `.env` (Task 2); compose.yaml sources password only via `${POSTGRES_PASSWORD:-...}`, no literal secret (Task 1) |
| T-01-02 | Elevation of Privilege | Postgres reachable on host network | mitigate | `#ports: 5432` block kept commented out; Coder reaches DB by service name `database` only (Task 1) |
| T-01-03 | Tampering | Coder image drift via `:latest` | mitigate | Pin default to `v2.33.8`; `CODER_VERSION`/`CODER_REPO` keep upgrades explicit and overridable (Task 1) |
| T-01-04 | Information Disclosure | `*.try.coder.app` dev tunnel active in production | mitigate | `CODER_ACCESS_URL` interpolated from `.env`; setting a real URL disables the tunnel (Task 1; operator docs in Plan 02) |
| T-01-05 | Information Disclosure | Postgres data world-readable on host | mitigate | Bind-mount dir owned `999:999` (documented chown prerequisite, OPS-03 in Plan 02); not world-readable by default |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs in this phase — only the official `ghcr.io/coder/coder` and `postgres:17` images are pulled (RESEARCH Package Legitimacy Audit: N/A) |
</threat_model>
