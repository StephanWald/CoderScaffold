<objective>
Fix the BAK-01 / SC-1 blocker: `scripts/backup.sh` exits 1 on every run at its integrity-check step.

The integrity check (current lines 98-101) runs `pg_restore --list /dev/stdin` with the dump piped via `docker compose exec -T database` stdin. `pg_restore --list` on a custom-format (-Fc) archive requires a SEEKABLE regular file (PostgreSQL 17 docs); `/dev/stdin` is a non-seekable stream, so the check fails for every valid dump. A secondary fault (docker/compose #8909) corrupts binary stdin into `exec -T`, especially on macOS Docker Desktop. The dump WRITE path is correct — only the verification is broken.

Purpose: After this fix lands, `./scripts/backup.sh` against a real stack exits 0 and reports `Backup complete`, satisfying SC-1 / BAK-01 while preserving the BAK-03 exit-code contract.
Output: Modified `scripts/backup.sh` whose integrity check copies the dump into the container as a seekable file, runs `pg_restore --list` against that file, removes it (even on failure), and surfaces pg_restore's real error on failure.

Scope: `scripts/backup.sh` ONLY. Do NOT touch `scripts/restore.sh` — its single forward-pass full `pg_restore` does not require seeking and is unaffected by this root cause.
</objective>
