## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Tampering / Denial (data loss) | startup_script `rm -rf "$HOME/.claude"` and `rm -f "$HOME/.claude.json"` | mitigate | Removal is gated behind `[ ! -L ... ]` (only acts on a real, non-symlink path) and is preceded by a no-clobber migration copy (`cp -an` / `cp -n`) into the shared volume, so a developer's prior config is preserved before deletion. Paths are string literals `$HOME/.claude` and `$HOME/.claude.json` — never a variable that could expand to `$HOME` or `/`, so the guard cannot be tricked into removing the wrong path. Once the symlink exists the guard is a no-op (no repeated destruction). |
| T-04-07 | Repudiation / Information (operator misjudgment) | README cleanup comment | mitigate | Comment reworded to state the true blast radius (permanent destruction of auth, settings, skills, MCP config), so an operator cannot delete the volume believing it is "not data-unsafe". |
