<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator host folder → coder build context | License-gated BBj JAR + certificate cross into the image build; must never enter git |
| workspace container → BLS license server | LICENSE_SERVER host:port; network trust to an external licensing service |
| workspace app (8888) → Coder proxy | BBjServices HTTP surface exposed via coder_app (subdomain routing) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-m12-01 | Information Disclosure | BBj*.jar / certificate.bls / license server | mitigate | Assets live only in the gitignored host folder + bind-mounted read-only; secrets (BBJ_LICENSE_SERVER) go in gitignored .env, only .env.example placeholders committed |
| T-m12-02 | Elevation of Privilege | coder_app share scope | mitigate | share = "owner" restricts the 8888 app to the workspace owner, not authenticated/public |
| T-m12-03 | Tampering | build context bind mount | accept | Mount is :ro (read-only); the provisioner cannot write back into the operator's asset folder |
| T-m12-04 | Denial of Service | BBjServices background launch | mitigate | startup_script launch is non-fatal (WR-03) — a crashed/absent server never aborts the agent, so editors/SSH stay reachable |
| T-m12-SC | Tampering | terraform provider installs (init) | accept | Providers are pinned (coder ~> 2.18, kreuzwerker/docker ~> 4.4) matching the audited java-fullstack fork; no new package-manager installs introduced |
