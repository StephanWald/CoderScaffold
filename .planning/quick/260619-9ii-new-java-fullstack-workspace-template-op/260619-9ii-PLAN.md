# Quick Task 260619-9ii — java-fullstack workspace template

A new, universal Coder workspace template for the user's stack: **Java/Spring Boot
backends + JavaScript/TypeScript UIs** (e.g. webforJ).

## Requirements (from user)

1. Optional **git repo clone** parameter, prompted at workspace creation.
2. Linux container with a **selectable JDK**: Adoptium Latest 21 & 25, plus Oracle
   JDK 21 & 25 (Oracle optional — feasible without login under NFTC).
3. **Maven**.
4. **Node** for JS/TS development (e.g. with webforJ).

## Decisions (confirmed with user)

- JDK strategy: **build-time single JDK** — a `coder_parameter` selects which JDK
  is installed; the value is a Docker build arg, so each selection builds/caches
  its own image and changing it rebuilds.
- **Include Oracle 21 + 25** — all four options offered.
- **Maven 3.9.x** (pinned 3.9.16; Maven 4 still RC, not production-safe).
- Defaults (stated, not vetoed): base on the existing `templates/docker` pattern
  (persistent home volume, code-server, JetBrains/IntelliJ, claude-code module,
  webforJ MCP preconfig, GSD); Node LTS via NodeSource; editors open the cloned
  repo when `git_repo` is set, else `/home/coder`.

## Implementation

New `templates/java-fullstack/` (auto-discovered by `scripts/push-templates.sh`):

- **`main.tf`** — modeled on `templates/docker/main.tf`, adding:
  - `data "coder_parameter" "git_repo"` (optional, mutable=false, order 1)
  - `data "coder_parameter" "jdk"` (4 options, default adoptium-21, mutable=true)
  - `var.maven_version` (default `3.9.16`)
  - `locals` deriving `project_folder` from the repo URL (basename minus `.git`)
  - `docker_image` name + `triggers` include the JDK value; `build_args` pass
    `BASE_IMAGE`, `JDK`, `MAVEN_VERSION`
  - agent `env` exports `JAVA_HOME=/opt/java/default`, `MAVEN_HOME=/opt/maven`
  - startup_script clone block (idempotent, non-fatal) using the parameter
  - editors' `folder = local.project_folder`
- **`Dockerfile`** — `BASE_IMAGE` + Node LTS + build-time JDK (arch-detected
  Adoptium API / Oracle latest URLs) installed to `/opt/java/default` via
  `update-alternatives`, + pinned Maven to `/opt/maven` (symlinked to
  `/usr/local/bin/mvn`), + `/etc/profile.d` exports. All system-wide (NOT in
  `/home/coder`, which the persistent volume shadows).

## Verify

- `terraform fmt -check` + `terraform validate` (with providers) — clean.
- JDK/arch URL selection resolves correctly for all 4 JDKs × {amd64,arm64};
  error paths fire for unknown JDK/arch. All 8 URLs pre-confirmed HTTP 206.
- `docker build --check` — no warnings.
- **Live gate** (per "infra needs a live deploy gate"): real `docker build` of
  the adoptium-21 variant; verify `java -version` and `mvn -v` in the image.
- README documents the new template.

## Done

`templates/java-fullstack/` builds a working image with the selected JDK + Maven +
Node, clones an optional repo, and exposes VS Code + IntelliJ + Claude Code; pushed
via the existing bulk script.
