# Quick Task 260619-93j

Preconfigure the Claude Code setup in the workspace containers to include the
hosted webforJ MCP server (`https://mcp.webforj.com/`) so every provisioned
workspace has it registered out of the box.

## Background

Both workspace templates wire Claude Code per-owner through the shared
`~/.claude.json`, which is symlinked into the per-owner shared Docker volume and
managed in `coder_agent.main.startup_script`:

- `templates/docker/main.tf`
- `templates/coderscaffold/main.tf`

Claude Code reads user-scope MCP servers from the top-level `mcpServers` key of
`~/.claude.json`. Injecting the webforJ entry there means it is preconfigured for
every workspace the owner starts, and (because the file lives on the shared
volume) it survives workspace stop/start/delete alongside the owner's auth.

## Task

In each template's `startup_script`, immediately AFTER the line:

```sh
ln -sf "$CLAUDE_SHARED/dot-claude.json" "$HOME/.claude.json"
```

and BEFORE the `# ── GSD (gsd-core) ...` install block, add an idempotent step
that merges a user-scope MCP server entry into `$CLAUDE_SHARED/dot-claude.json`:

- Use `node` (guaranteed present — Node.js/npm is layered into the workspace
  image) to read-merge-write the JSON so all existing config/auth is preserved.
- Ensure `cfg.mcpServers.webforj = { type: "http", url: "https://mcp.webforj.com/" }`.
- Write back with `JSON.stringify(cfg, null, 2)`.
- Idempotent: safe to re-run on every workspace start (sets the entry each time,
  keeping the URL correct without duplicating).
- Guard with `command -v node`; if node is absent, warn-and-continue.
- Non-fatal under `set -e` (follow the existing WR-03 warn-and-continue pattern).
- Add a short explanatory comment block matching the surrounding style.

Keep both templates byte-for-byte consistent in this block.

## Verify

- `terraform fmt -check` (or visual diff) — block is valid HCL heredoc content.
- The injected `node -e` script is valid: parses `{}`-or-existing JSON, sets the
  nested key, writes pretty JSON + trailing newline.
- Both templates contain the identical block in the same location.

## Done

Both `templates/docker/main.tf` and `templates/coderscaffold/main.tf` register the
webforJ MCP server in the shared `~/.claude.json` on workspace start, idempotently
and non-fatally, with all prior config preserved.
