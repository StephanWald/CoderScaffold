# Quick Task 260619-93j — Summary

Preconfigured the Claude Code setup in the workspace containers to include the
hosted webforJ MCP server (`https://mcp.webforj.com/`).

## What changed

Added an idempotent, non-fatal startup step to both workspace templates'
`coder_agent.main.startup_script`, immediately after the `~/.claude.json` symlink
is created and before the GSD install block:

- `templates/docker/main.tf`
- `templates/coderscaffold/main.tf`

The step uses `node` (guaranteed in the workspace image — Node.js is layered on)
to read-merge-write the shared `~/.claude.json`, ensuring:

```json
"mcpServers": { "webforj": { "type": "http", "url": "https://mcp.webforj.com/" } }
```

Because `~/.claude.json` lives on the per-owner shared Docker volume, the entry is
preconfigured for every workspace the owner starts and persists across workspace
stop/start/delete alongside their auth.

## Properties

- **Preserves existing config/auth** — read-merge-write, never overwrites the file
  wholesale. Verified: an existing top-level key and a second MCP server both
  survive the merge.
- **Idempotent** — re-asserts the same entry on every start; re-running yields
  exactly one `webforj` entry (verified). Keeps the URL correct over time.
- **Robust to first run** — produces valid JSON even when the file is missing or
  empty (verified).
- **Non-fatal under `set -e`** — guarded by `command -v node`; a missing node or a
  write failure logs a `WARN` and continues (existing WR-03 pattern), never
  aborting `startup_script`.
- **Both templates identical** — the injected block is byte-for-byte the same in
  both files (verified via diff).

## Verification

- Embedded `node -e` merge logic tested in isolation: preserves prior config +
  other MCP servers, idempotent (single entry), valid output from a missing file.
- `terraform fmt -check` — both templates clean (also fmt-normalized a pre-existing
  unrelated `env` alignment in `templates/docker/main.tf` and matched it in
  `templates/coderscaffold/main.tf` so the two stay consistent).
- Block-consistency diff between the two templates: identical.

## Notes / follow-ups

- The transport is declared as `type: "http"` (Claude Code streamable-HTTP MCP).
  If `mcp.webforj.com` only speaks SSE, change `http` → `sse` in both blocks.
- Not yet validated against a live workspace deploy (consistent with the standing
  "infra needs a live deploy gate" caveat) — the static checks above cover the
  merge logic and HCL validity, but end-to-end registration should be confirmed on
  a real workspace start.
